# בנאי האתר ו-CMS — מדריך הפעלה

מערכת לעריכת **אתר השיווק של Robert** מתוך פאנל האדמין, ללא קוד. כל התוכן, העיצוב,
הסקשנים, הבאנרים, הבלוג והמדיה נשמרים במסד הנתונים ומתעדכנים באתר החי מיידית.

> עיקרון-על: לא נגענו בעיצוב. מחלקות ה-CSS המקוריות (`landing.module.css`) נשארו זהות —
> רק התוכן מוזרם אליהן ממסד הנתונים. האתר נראה זהה לחלוטין אחרי המעבר.

## הפעלה ראשונית (פעם אחת)

1. **הרצת מיגרציה** — ב-Supabase SQL Editor, הרץ את
   [app/supabase/migrations/0006_website_builder.sql](app/supabase/migrations/0006_website_builder.sql)
   (אחרי 0001–0005). זה יוצר את כל הטבלאות + ה-bucket `site-media` + עמודת `admin_role`.
2. **כניסה לפאנל** → "בנאי האתר" → "עמודים" → לחיצה על **"אתחול ראשוני"**.
   זה מייצר את דף הבית מהתוכן הקיים, ערכת עיצוב ברירת מחדל, והגדרות (כותרת/תחתית).
3. זהו — האתר עכשיו נשלט ממסד הנתונים. ערוך מה שתרצה ולחץ **"פרסם"**.

## תהליך העבודה: טיוטה → תצוגה מקדימה → פרסום

- כל עריכה נשמרת כ**טיוטה** (לא משפיעה על האתר החי).
- **"תצוגה מקדימה"** פותח את האתר עם הטיוטה (Draft Mode) — רק אתה רואה.
- **"פרסם"** מעתיק טיוטה → פרסום, שומר גרסה בהיסטוריה, ומרענן את האתר מיידית.

## המסכים (פאנל אדמין → בנאי האתר)

| מסך | מה עושים |
|---|---|
| עמודים | דף הבית, עמודים וכתבות. עריכת סקשנים: הוספה/הסרה/סדר/שכפול/הסתרה/תזמון + SEO + גרסאות + Undo/Redo |
| עיצוב | צבעים, פונטים (Google Fonts), פריסה, מצב כהה — חל מיד על האתר |
| ערכות נושא | שמירה/שכפול/ייבוא/ייצוא/הפעלה של ערכות עיצוב |
| מדיה | העלאת תמונות/וידאו ל-Supabase Storage, העתקת כתובת |
| בלוג | כתבות (עורך טקסט עשיר), קטגוריות, כותבים, SEO לכל כתבה |
| באנרים ופופאפים | פופאפ קבלת פנים/יציאה, באנרים צפים — עם תזמון וטריגרים |
| הגדרות ושיווק | כותרת, תחתית, פס הכרזה, וידג'ט וואטסאפ, רשימת ניוזלטר |
| אנליטיקס | צפיות/סשנים/המרות/עמודים מובילים/מקורות (גרפים) |
| יומן וגיבוי | יומן שינויים מלא, ייצוא/שחזור גיבוי, שחזור ברירת מחדל |
| קוד מותאם | CSS/JS/סקריפטים (מנהל ראשי בלבד) |
| צוות והרשאות | ניהול הרשאות צוות הניהול |

## הרשאות (admin_role)

- **מנהל ראשי (super_admin)** — הכל, כולל קוד מותאם, צוות, גיבוי.
- **מנהל (admin)** — תוכן, עיצוב, הגדרות.
- **עורך (editor)** — תוכן בלבד.
- **תמיכה (support)** — צפייה בלבד.

מנהלים קיימים שודרגו אוטומטית ל-super_admin. הפריטים בתפריט מוסתרים לפי הרשאה.

## ארכיטקטורה (למפתחים)

- **מודל תוכן = מסמכי JSON.** לכל עמוד `draft_doc`/`published_doc` עם מערך `sections`.
  הגדרות גלובליות באותו מבנה ב-`site_settings`.
- **רינדור ציבורי:** [app/src/app/page.tsx](app/src/app/page.tsx) ו-[app/src/app/[slug]/page.tsx](app/src/app/[slug]/page.tsx)
  הם Server Components שקוראים תוכן מפורסם (cached, `revalidateTag('site')`) דרך
  [app/src/lib/site/content.ts](app/src/lib/site/content.ts), ומרכיבים סקשנים דרך
  [app/src/components/sections/SectionRenderer.tsx](app/src/components/sections/SectionRenderer.tsx).
- **עיצוב:** [app/src/components/site/ThemeStyle.tsx](app/src/components/site/ThemeStyle.tsx)
  מזריק את משתני ה-CSS של הערכה הפעילה (מנצח את `.landing` בעזרת `[data-marketing][data-rb-theme]`).
- **מצב דמו:** ללא Supabase, האתר מרונדר מ-[app/src/lib/site/defaults.ts](app/src/lib/site/defaults.ts)
  (מקור האמת שגם מזריע את ה-DB) — כך האתר אף פעם לא ריק.
- **מולטי-טננט:** כל טבלה נושאת `site_id`. כיום אתר אחד (`is_primary`), המודל תומך ברבים.
- **API:** `app/src/app/api/admin/site/*` (מאומת `requireAdmin` + `requirePermission`),
  ציבורי `app/src/app/api/site/{track,newsletter}`.

## אבטחה

- קוד מותאם (CSS/JS/סקריפטים) — מנהל ראשי בלבד; סקשני HTML מותאם עוברים סינון (`sanitize.ts`).
- RLS על כל הטבלאות (`is_admin()`); קריאות ציבוריות עוברות דרך service-role לתוכן מפורסם בלבד.
- נקודות הקצה הציבוריות מוגבלות בקצב ומאומתות.
