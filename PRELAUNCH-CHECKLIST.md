**הערה חשובה על "אימות פונקציונלי חי":** פאנל האדמין מוגן ב-2FA (סעיף 4 ב-
`CLAUDE.md`/`admin-auth.ts`) — סוכן אוטומטי **לא יכול ולא אמור** לעקוף אותו. בנוסף,
סביבת הפיתוח המקומית (`.env.local`) מחוברת ל-**אותו** Supabase/Twilio/Resend של
הפרודקשן (אין סביבת staging נפרדת) — כך שכל פעולת כתיבה חיה (הפעלת תחזוקה, איפוס
וואטסאפ אמיתי, שליחת מייל אמיתי) הייתה משפיעה על נתונים/משתמשים אמיתיים. לכן הסעיפים
1-4 אומתו **בקוד + build + טסטים בלבד** בסשן הזה, ונדרש מעבר ידני קצר של הבעלים
(מפורט למטה) לפני הסתמכות מלאה עליהם.

### מה הבעלים צריך לבדוק ידנית (כ-10 דקות, אחרי כניסה לפאנל עם 2FA)
- [ ] **וואטסאפ:** לחבר בוט בדיקה דרך OTP, לוודא שהבאדג' עובר 🟡 מתחבר → 🟢 מחובר.
      לנתק ולוודא 🔴. (אין מספר בדיקה? אפשר לדלג ולסמוך על בדיקת קוד + build.)
- [ ] **תחזוקה:** להפעיל מצב תחזוקה מ-`/admin/system`, לוודא ב־חלון גלישה פרטית ש-
      `/login`, `/reset-password`, `/cancel`, `/preview`, ודף `[slug]` ציבורי — כולם
      מפנים ל-`/maintenance`. לוודא שפאנל האדמין עצמו **נשאר נגיש**. לכבות בחזרה.
- [ ] **ניהול משתמשים:** בעמוד `/admin/users/[id]` של משתמש בדיקה — ללחוץ "אפס חיבור
      וואטסאפ" ולוודא מופיע ב-timeline; לשלוח הודעת בדיקה ולוודא הגעה + רישום audit
      (בלי גוף ההודעה, רק הנושא — בכוונה).
- [ ] **סטטוס מנוי:** לפתוח משתמש עם `is_comp=true` (או להעניק זמנית) ולוודא שבדשבורד
      הלקוח **ובעמוד הפרטים באדמין** המילה "הענקה" לא מופיעה בשום מקום — רק "מסלול
      X" + תג משני "הטבה פעילה".

## 2. פעולות בעלים מחוץ ל-scope הקוד (חוסמות אמינות שליחה בפרודקשן)

| # | פעולה | למה חוסם | איפה |
|---|---|---|---|
| 1 | **Meta Embedded Signup** — להשלים `META_APP_ID`, `META_APP_SECRET`, `META_CONFIG_ID` ב-Vercel production | אומת (`vercel env ls production`): שלושתם **חסרים** היום. נתיב ה-connect-meta כתוב ומוכן בקוד אך חשוך לגמרי בפרודקשן. | Meta Business Manager → Vercel → Settings → Environment Variables |
| 2 | **או**, לחלופין: לוודא ב-Twilio Console שהמספר המחובר דרך OTP באמת רשום ומאושר כ-WhatsApp Sender | בלי זה, ה-DB יכול לומר "מחובר" בעוד הודעות בפועל לא יוצאות — זו בדיוק הבעיה שפתחה את התוכנית הזו. | Twilio Console → Messaging → WhatsApp Senders |
| 3 | **Twilio Account SID בפרודקשן** — לוודא שה-SID הפעיל בפרודקשן תואם את המספר המאושר (פריט פתוח, לא חוסם, מתועד גם ב-memory `product_improvement_12_program`) | חוסר-התאמה יגרום לשליחות כושלות שקטות | Vercel Environment Variables → `TWILIO_ACCOUNT_SID` |

לפחות **אחד** מבין 1/2 חובה לפני הבטחת "וואטסאפ עובד" ללקוח משלם.

## 3. מיגרציית DB — 0014

- **קובץ:** `app/supabase/migrations/0014_wa_connection_status.sql` — מוכן, **גדור בקוד**
  (הערת `GATED` בראש הקובץ), מוסיף `wa_connection_status`/`wa_last_error`/
  `wa_connected_at` ל-`bots` + backfill למחוברים קיימים.
- **סטטוס:** **לא הוחלה** בסשן הזה (לא ניתן `APPROVED - DATABASE`). לא ניתן היה גם
  לאמת מול הפרויקט החי דרך MCP — טוקן Supabase לא מוגדר בסביבה המקומית (ראה
  `.mcp.json`/`settings.local.json`).
- [ ] **לפני deploy:** הבעלים לאשר `APPROVED - DATABASE`, להריץ את הקובץ ב-Supabase
      SQL editor (או דרך MCP אם הטוקן יוגדר), ואז לוודא ב-`list_migrations`/בדיקת
      עמודות שהיא נקלטה. הקוד **תומך בהיעדרה** (כתיבות ה-status הן best-effort,
      נכשלות בשקט אם העמודות לא קיימות) — כך שאין חובה טכנית-דחופה, אבל בלעדיה
      הבאדג'ים בדשבורד/אדמין תמיד יראו "לא מחובר" ברירת מחדל.
